# Thesis Risk ML Pipeline

## Project Overview
This repository contains the full reproducible implementation for the thesis:

**Predicting Project Delays and Cost Overruns Using Machine Learning: Towards an AI-Driven Early Risk Detection Framework**

The project develops a machine learning pipeline to predict:
- **cost overrun (%)**
- **schedule overrun (%)**

using synthetic **planning-phase project data** generated through Monte Carlo simulation.

The repository includes:
- synthetic data generation
- feature engineering
- preprocessing and leakage-safe model pipelines
- regression model training and comparison
- early-warning risk detection analysis
- diagnostic and explainability outputs
- thesis-ready tables and figures

The implementation is designed to be:
- reproducible
- thesis-oriented
- modular
- transparent
- easy to inspect through notebooks and scripts

## Setup / Installation
1. Clone the repository:

2. Create and activate a virtual environment (recommended):

3. Install dependencies:

## Quick Start
Run the full pipeline end-to-end:

```bash
python run_pipeline.py
```

This will:
- generate the synthetic dataset
- train and evaluate the regression models
- export tables, figures, and saved models

Note: the pipeline writes working outputs to `outputs/tables/` and `outputs/figures/`. The repository also includes curated thesis artifacts in `outputs/thesis_tables/` and `outputs/thesis_figures/`, plus archived exploratory artifacts in `outputs/exploratory/`.

## Reproducibility Note
All paths are configured relative to the repository root using `pathlib.Path`.

This means the project does not depend on machine-specific absolute paths and should run correctly from a local folder or a GitHub clone, provided the dependencies are installed.

Reproducibility is supported by:
- fixed random seeds
- repository-relative paths
- saved configuration outputs
- saved datasets, tables, figures, and trained model artifacts

The notebooks provide transparent step-by-step inspection of the full workflow for academic review.

## Notebooks Overview
- `notebooks/01_synthetic_data_creation.ipynb`
  Walkthrough of synthetic data generation, tier structure, missingness, outlier simulation, and feature engineering.

- `notebooks/02_model_creation.ipynb`
  Walkthrough of preprocessing, train/test split, cross-validation, model tuning, selection, and saved model creation.

- `03_model_review.ipynb`
  Review notebook for loading saved models, comparing predictive performance, generating thesis tables/figures, inspecting feature importance, and checking diagnostics.

## Implementation Notes
The code preserves the thesis methodology and modeling logic:
- planning-phase variables only
- leakage-safe preprocessing inside CV pipelines
- dual regression targets for cost and schedule overruns
- early-warning threshold evaluation
- bootstrap confidence intervals
- explainability outputs for tree-based models

The repository changes are limited to reproducibility and packaging improvements, not methodological changes.
